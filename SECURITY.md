# Security Policy

## Supported Versions

Security fixes are applied to the latest state of the repository on the `main` branch.

## Reporting a Vulnerability

If you discover a security issue, please report it privately.

1. Do not open a public GitHub issue.
2. Share details with maintainers, including:
   - Description of the vulnerability
   - Reproduction steps or proof of concept
   - Potential impact
   - Suggested mitigation, if available
3. Allow reasonable time for assessment and remediation before public disclosure.

## Response Process

Maintainers will:

- Acknowledge receipt of your report
- Assess severity and impact
- Prepare and test a fix
- Release the fix and communicate remediation guidance

## Scope Notes

Please include whether the issue affects:

- Backend API authentication and session handling
- Environment secret handling
- MongoDB access control and data exposure
- Frontend token or request handling
